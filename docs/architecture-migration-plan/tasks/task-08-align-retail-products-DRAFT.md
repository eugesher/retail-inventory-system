## Goal

Reshape the products module of `apps/retail-microservice/src/` into the
per-module hexagonal layout used by notification and inventory. Domain
model `Product` plus a `Price` value object; use cases for
Create / Update / Find / List; `ProductRepositoryPort` →
`ProductTypeormRepository`; `ProductEventsPublisherPort` →
`ProductRabbitmqPublisher`. No code under retail/orders is touched in
this task.

## Steps

1. **Inventory the products code.** List every file under
   `apps/retail-microservice/src/` that belongs to the products
   domain — controllers, services, entities, DTOs. (verify in
   task-01: confirm the actual folder name; the existing repo uses
   `app/api/<feature>/` so it might be `app/api/products/`.)

2. **Create the module skeleton** at
   `apps/retail-microservice/src/modules/products/{application,domain,infrastructure,presentation}/`.

3. **Domain.**
   - `domain/product.model.ts` — pure class. Constructor enforces
     invariants (price > 0, name non-empty, etc.).
   - `domain/price.value-object.ts` — extends `ValueObjectBase` from
     `libs/ddd`. Holds amount + currency, validates currency code.
   - `domain/events/product-created.event.ts`,
     `product-updated.event.ts` — extend `DomainEventBase`.

4. **Application.**
   - `application/ports/product.repository.port.ts` — interface
     `ProductRepositoryPort` (`findById`, `findBySku`, `save`,
     `softDelete`, `list`) + DI symbol `PRODUCT_REPOSITORY`.
   - `application/ports/product-events.publisher.port.ts`.
   - `application/use-cases/`:
     `create-product.use-case.ts`, `update-product.use-case.ts`,
     `find-product.use-case.ts`, `list-products.use-case.ts`.
   - `application/dto/`: `create-product.command.ts`,
     `update-product.command.ts`, `find-product.query.ts`,
     `product.view.ts`.

5. **Infrastructure.**
   - `infrastructure/persistence/product.entity.ts` — TypeORM entity.
     Relocate from existing `common/entities/` (verify in task-01).
   - `infrastructure/persistence/product.mapper.ts`.
   - `infrastructure/persistence/product-typeorm.repository.ts`.
   - `infrastructure/messaging/product-rabbitmq.publisher.ts`.
   - `infrastructure/cache/product-redis.cache.ts` — placeholder if
     cache wiring is deferred to task-11.
   - `infrastructure/products.module.ts` — wires ports to adapters,
     `DatabaseModule.forFeature([Product])`, `MessagingModule`,
     `CacheModule`.

6. **Presentation.**
   - `presentation/products.controller.ts` —
     `@MessagePattern('retail.product.*')` handlers (verify in
     task-01: existing pattern names need migrating; CLAUDE.md
     mentions `RETAIL_ORDER_*` pattern names but no
     `RETAIL_PRODUCT_*` — confirm whether products are exposed to
     the gateway).
   - `presentation/dto/`: any HTTP/RPC-only DTOs.

7. **Wire** the module into `app.module.ts`. Drop the old
   products code path.

8. **Tests.**
   - Unit tests against in-memory `ProductRepositoryPort`.
   - Mapper round-trip test (entity ↔ domain).
   - Integration test against testcontainers MySQL for the
     TypeORM adapter.

9. **Delete** the old products code path once every consumer is
   repointed.
