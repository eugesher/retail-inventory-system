## Goal

Reshape `apps/inventory-microservice/src/` from its current
`app/api/<feature>/providers/*-<action>.service.ts` layout (verify in
task-01) into per-module
`{application,domain,infrastructure,presentation}` directories. The
existing `*-<action>.service.ts` files map cleanly to use cases — the
work is mostly relocation plus introducing ports/adapters and a
domain model that does not double as the TypeORM entity.

## Steps

1. **Inventory the service.** Per `apps/inventory-microservice/src/`,
   list every controller, service, entity, DTO, and module. Note
   which feature they belong to (stock, warehouses, reservations —
   verify in task-01 which exist).

2. **Create module skeletons.** For each feature, create
   `modules/<feature>/{application/{use-cases,ports,dto},domain/{events},infrastructure/{persistence,messaging,cache},presentation/dto}`.

3. **Per-module work for `stock`** (template — repeat for
   `warehouses` and `reservations` if they exist):

   a. **Domain.**
   - `domain/stock-item.model.ts` — pure class. Constructor
     enforces invariants (`quantity >= 0`, `reservedQuantity <= quantity`).
   - `domain/events/stock-reserved.event.ts`,
     `stock-low.event.ts` — extend `DomainEventBase` from
     `libs/ddd`.

   b. **Application.**
   - `application/ports/stock.repository.port.ts` — interface
     `StockRepositoryPort` with `findById`, `findBySku`, `save`,
     etc., plus DI symbol `STOCK_REPOSITORY`.
   - `application/ports/stock-events.publisher.port.ts` — interface
     for emitting domain events to RabbitMQ.
   - `application/use-cases/`:
     `reserve-stock.use-case.ts`, `release-stock.use-case.ts`,
     `adjust-stock.use-case.ts`, `get-stock.use-case.ts`,
     `list-stock.use-case.ts`. Each is a thin class with one
     `execute(cmd | query)` method. Map from existing
     `*-<action>.service.ts` files.
   - `application/dto/`: command + query + view DTOs per the naming
     convention.

   c. **Infrastructure.**
   - `infrastructure/persistence/stock-item.entity.ts` — TypeORM
     `@Entity()` (relocate from existing `common/entities/`).
   - `infrastructure/persistence/stock-item.mapper.ts` — entity ↔
     domain.
   - `infrastructure/persistence/stock-typeorm.repository.ts` —
     implements `StockRepositoryPort`. Extends
     `BaseTypeormRepository` from `libs/database`.
   - `infrastructure/messaging/stock-rabbitmq.publisher.ts` —
     implements the events publisher port; uses routing keys from
     `libs/messaging`.
   - `infrastructure/cache/stock-redis.cache.ts` — wraps the
     `CachePort` from `libs/cache` for use by `GetStockUseCase`.
     (Cache wiring may be deferred to task-11; if so, leave a
     placeholder file with a TODO.)

   d. **Presentation.**
   - `presentation/stock.controller.ts` —
     `@MessagePattern('inventory.stock.*')` handlers (verify in
     task-01: existing pattern names may need migrating). Use cases
     are DI-injected.

4. **Wire the module** — `infrastructure/stock.module.ts` binds
   port symbols to adapters, imports `DatabaseModule.forFeature([...])`,
   `MessagingModule`, `CacheModule`, `LoggerModule`.

5. **Repeat 3–4 for any other inventory feature** found in step 1.

6. **Update `app.module.ts`** to import the new feature modules and
   drop references to the old `app/api/<feature>/` layout.

7. **Update `main.ts`** so the first import is the observability
   tracer.

8. **Move TypeORM migrations** (under `migrations/`) — only if any
   migration sits inside an app folder. The repository's
   `migrations/config/data-source.ts` points to a top-level
   `migrations/` directory; do not relocate that.

9. **Tests.**
   - Unit tests against in-memory `StockRepositoryPort` for invariant
     enforcement (e.g., reserving more than available throws).
   - Integration test against testcontainers MySQL for
     `StockTypeormRepository` (`yarn test:e2e` already provisions
     MySQL via `test:infra:up`; verify the existing `test/` setup
     can run integration-style tests against a real DB — verify in
     task-01).
   - Keep existing e2e tests passing.

10. **Delete the old `app/api/` folder** for inventory once every
    consumer is repointed.
