## Goal

End up with a complete, consistent, indexable ADR set under
`docs/adr/`. Every architectural decision the migration encoded —
monorepo layout, hexagonal per-service, TypeORM-as-persistence,
RabbitMQ-as-bus, Redis cache-aside, Pino+OTel observability,
JWT/RBAC at the gateway (only if in scope), eslint-plugin-boundaries
as the architecture lint, the contracts/ddd/database lib split — is
either already an ADR or is added in this task. ADR-001
(`record-architecture-decisions`) describes the format and points at
MADR or Nygard.

## Steps

1. **Audit existing ADRs.** List every file under `docs/adr/`. For
   each, note: number, title, status, date, decision summary. Save
   the audit in a table in `_carryover-13.md`.

2. **Identify gaps.** Compare the audit against the recommendation's
   Section 6 ADR list:
   - 0001 / 001 — Record architecture decisions (likely already from
     task-01).
   - 0002 / 002 — Monorepo with apps and libs.
   - 0003 / 003 — Hexagonal architecture per service.
   - 0004 / 004 — TypeORM + MySQL as persistence.
   - 0005 / 005 — RabbitMQ as the message bus.
   - 0006 / 006 — Redis cache-aside (already exists as 002 — verify
     in task-01 whether it gets renumbered or kept as-is).
   - 0007 / 007 — Pino + OpenTelemetry (existing 001 — same caveat).
   - 0008 / 008 — JWT + RBAC at the gateway (only if in scope).

   Per-task work likely covered 003, 004, 005, 007 (and arguably 008)
   already. This task fills whichever are still missing.

3. **Add missing ADRs.** Each ADR uses the project's chosen format
   (MADR or Nygard, decided in task-01) with sections: Status,
   Context, Decision, Consequences. Status is "Accepted" with the
   date of writing.

4. **Reconcile numbering.** If task-01 standardized on a different
   width than the existing 001/002 (e.g., switched to 4-digit), this
   task either renumbers or updates references. **Do not silently
   leave a mix.**

5. **Write `docs/adr/index.md`** (or `0000-index.md`) — a flat list:
   number → title → status → one-line summary. This file is the
   reading order for any new contributor.

6. **Cross-reference.** Each ADR that depends on another links to it
   ("see ADR-NNN for the per-service hexagonal layout"). Do not
   restate decisions across ADRs — link instead.
