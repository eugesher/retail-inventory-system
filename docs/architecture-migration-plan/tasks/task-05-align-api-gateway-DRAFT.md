## Goal

Convert `apps/api-gateway/src/` from its current
`app/api/<feature>/` flat layout into
`src/modules/<feature>/{application,domain,infrastructure,presentation}/`
per the recommendation. The gateway is mostly a presentation layer (it
delegates to other services via RabbitMQ), so its `domain/` and
`application/` folders are slim — but the structure must still be in
place so the lint rules in task-12 can enforce it uniformly.

## Steps

1. **Inventory `apps/api-gateway/src/`.** List every controller,
   service, DTO, guard, strategy, module — and note which feature
   (auth / retail-proxy / inventory-proxy / health) each belongs to.

2. **Create the modules skeleton.**
   `apps/api-gateway/src/modules/{retail,inventory}/{application,domain,infrastructure,presentation}/`.
   Add `auth/` under `modules/` only if `libs/auth` is in scope.

3. **Move proxy controllers** for retail and inventory to
   `modules/<svc>/presentation/<svc>.controller.ts`. Each proxy
   controller's HTTP routes remain the same; only the file path
   moves and the imports update.

4. **Define gateway ports** in
   `modules/retail/application/ports/retail-gateway.port.ts` and
   `modules/inventory/application/ports/inventory-gateway.port.ts`.
   Each port exposes the methods the controller needs (e.g.,
   `createOrder(cmd)`, `confirmOrder(cmd)`, `getOrder(id)` for retail).

5. **Implement adapters** in
   `modules/<svc>/infrastructure/messaging/<svc>-rabbitmq.adapter.ts`.
   Each adapter holds the injected `ClientProxy` from
   `libs/messaging` and turns port calls into `client.send()` /
   `client.emit()` invocations. Routing keys and correlation IDs come
   from `libs/messaging` and `libs/observability`.

6. **Replace controllers' direct `client.send()` calls** with
   port-method invocations. The controllers must no longer import
   `ClientProxy` directly — they take the port symbol from DI.

7. **Replace inline DTOs with imports from `libs/contracts`.** Any
   DTO that crosses the gateway/microservice boundary must come from
   `libs/contracts`. DTOs that are gateway-internal (HTTP-only) stay
   in the module's `presentation/dto/`.

8. **Migrate auth (if in scope).** Move
   `auth/jwt.strategy.ts` → `modules/auth/infrastructure/jwt/jwt.strategy.ts`,
   delegating token issuance to a `TokenPort`
   (`modules/auth/application/ports/token.port.ts`). Implement
   `JwtTokenAdapter` in `modules/auth/infrastructure/jwt/`.
   Move `auth.controller.ts` → `modules/auth/presentation/`.
   Split `auth.service.ts` into `LoginUseCase`, `RefreshTokenUseCase`,
   `ValidateUserUseCase`. Re-export `RolesGuard` from `libs/auth`
   rather than duplicating it. (Skip this entire step if auth is not
   in scope.)

9. **Wire the request-correlation middleware** from
   `libs/observability` in `app.module.ts` (it must run before any
   route handler).

10. **Make `import '@retail-inventory-system/observability/tracer'`
    the first line of `apps/api-gateway/src/main.ts`** (or
    `@app/observability/tracer` per the chosen alias). The order
    matters — OTel must be initialized before any instrumented module
    loads. (verify in task-01: confirm the alias prefix.)

11. **Run e2e tests** for the gateway routes that already have e2e
    coverage (verify in task-01: list which routes do).
