## Goal

Stand up `apps/notification-microservice/` with the canonical hexagonal
layout: a `notifications` module with `domain/`, `application/`
(use cases + `NotifierPort`), `infrastructure/` (consumers + delivery
adapters), and `presentation/` (health/admin only). At least one
adapter (a logging notifier) is wired and unit-tested. This service
will be the **template** that tasks 07/08/09 reference when reshaping
inventory and retail.

## Steps

1. **Create the module skeleton.**
   `apps/notification-microservice/src/modules/notifications/{application,domain,infrastructure,presentation}/`.

2. **`domain/notification.model.ts`** — pure class with the fields a
   notification needs (recipient, channel, subject, body, metadata).
   No `@nestjs/*`. Validation via the value-object base from
   `libs/ddd`.

3. **`application/ports/notifier.port.ts`** — interface
   `NotifierPort { send(message: NotificationMessage): Promise<void> }`,
   plus a string DI symbol `NOTIFIER`.

4. **`application/use-cases/`** — at least:
   - `send-order-notification.use-case.ts` — consumes a
     `RetailOrderCreatedEvent` (from `libs/contracts/retail`) and
     emits a notification through `NotifierPort`.
   - `send-low-stock-alert.use-case.ts` — consumes
     `InventoryStockLowEvent` (from `libs/contracts/inventory`).

5. **`infrastructure/delivery/`** — adapters implementing
   `NotifierPort`:
   - `log.notifier.adapter.ts` — writes to Pino at `info` level. This
     is the default binding.
   - `email.notifier.adapter.ts` — class scaffold with a TODO; **do
     not** add `nodemailer` as a dep yet.
   - `webhook.notifier.adapter.ts` — class scaffold with a TODO.

6. **`infrastructure/consumers/`** — RabbitMQ subscribers:
   - `order-events.consumer.ts` — `@EventPattern('retail.order.created')`,
     invokes `SendOrderNotificationUseCase`.
   - `inventory-events.consumer.ts` —
     `@EventPattern('inventory.stock.low')`, invokes
     `SendLowStockAlertUseCase`.
   Pattern strings come from `libs/messaging` constants — no string
   literals.

7. **`infrastructure/notifications.module.ts`** — wires `NOTIFIER`
   symbol → `LogNotifierAdapter`, registers the consumers, imports
   `MessagingModule`, `LoggerModule`.

8. **`presentation/health.controller.ts`** — minimal health check
   responding to a `notification.health` message pattern (or HTTP
   `/health` if the gateway proxies through; verify in task-01).

9. **Update `apps/notification-microservice/src/main.ts`** to:
   - First line: `import '@retail-inventory-system/observability/tracer';`
   - Use `NestFactory.createMicroservice` with the RabbitMQ config
     from `libs/messaging`.
   - Use `LoggerModule` from `libs/observability`.

10. **Tests:**
    - Unit tests for both use cases with an in-memory `NotifierPort`
      double.
    - Unit test for `LogNotifierAdapter` against Pino's test
      transport.
    - E2E test that publishes a fake `OrderCreated` event to
      RabbitMQ and asserts a log line containing the recipient.

11. **Update `docker-compose.yml`** so `notification-microservice`
    depends on `rabbitmq` (verify in task-01: it may already).
