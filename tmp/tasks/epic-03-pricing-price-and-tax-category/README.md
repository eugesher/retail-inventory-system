## Cross-epic coupling notes

- **`epic-01`** is a hard prerequisite. The new `pricing:write` permission code is seeded into the existing `admin` + `catalog-manager` roles (the registry from epic-01 task-01). The api-gateway pricing endpoints reuse epic-01's `PermissionsGuard` + `@RequiresPermission()`.
- **`epic-02`** is a hard prerequisite — this epic colocates inside `catalog-microservice` and modifies `Publish Product`. If epic-02 has not landed, the `pricing/` sibling module has no host application; the `Publish Product` hard-fail in task-04 has no target.
- **`epic-04`** is downstream. The inventory-microservice consumer of `catalog.variant.created` (auto-init `StockLevel`) is not impacted here. However, the inventory side does NOT consume `catalog.price.changed` — pricing events are sink-only into the future event-store (`epic-11`) and the api-gateway read path. Task-03 documents this explicitly so a future inventory contributor does not over-bind.
- **`epic-05`** is downstream. The `Cart → Order` walking-skeleton inherits `Order.currency` from the price snapshot at write-time; doc `07-currency-immutability-on-order.md` exists to seal that contract before epic-05 lands.
- **`epic-11`** is downstream. The `catalog.price.changed` + `catalog.price.scheduled` routing keys are added here, but the audit/event-store consumer for them is owned by epic-11; no consumer binding is added in this epic.
- **`epic-15`** owns every excluded pricing concern (discounts, MSRP, B2B contract pricing, multi-channel scope, tax rate computation, currency conversion). The Exclusions Register entries already exist under `epic-15`'s scope; this epic adds no new Exclusions docs.
